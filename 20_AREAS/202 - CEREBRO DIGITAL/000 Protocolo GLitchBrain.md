# Protocolo GLitchBrain

El **GLitchBrain** no es solo un conjunto de notas; es un ecosistema de conocimiento vivo, diseñado por **GlitchMan** para evolucionar junto a sus proyectos y aprendizajes. Este protocolo define cómo opera, se mantiene y se expande el sistema.

## 1. Arquitectura PARA-WIKI 2.0
El cerebro está organizado en cuatro pilares fundamentales, optimizados para la búsqueda semántica y la agilidad:

- **10_PROJECTS**: Notas activas con fecha de finalización (ej. ITB, GlitchCat).
- **20_AREAS**: Responsabilidades de mantenimiento continuo (ej. Tecnología, Salud).
- **30_RESOURCES**: Temas de interés y material de estudio (ej. GlitchBro, Libros).
- **40_ARCHIVE**: Conocimiento frío o proyectos completados.

## 2. Los 3 Pilares del Mantenimiento
Para evitar el desorden (entropía), el sistema se apoya en tres procesos clave:

1. **Austeridad Visual**: Prohibido el uso de emojis en nombres de archivos, carpetas o metadatos. El foco es el contenido.
2. **Normalización NFC**: Todos los archivos deben seguir el estándar de codificación Linux para evitar duplicados por tildes.
3. **Skill Registry**: El sistema utiliza habilidades de IA (Antigravity) para auditar, limpiar y conectar notas automáticamente.

## 3. Flujo de Operación (Skills)
- **Vault Gardener**: Mantiene la salud estructural, audita links y aplica la austeridad visual.
- **GlitchBro Ingestor**: Procesa el material de estudio (como el curso de Python) y lo integra en la zona de recursos.
- **Project Catalyst**: Gestiona los hitos y MOCs de los proyectos activos.
- **Semantic Oracle**: Realiza búsquedas profundas y encuentra conexiones no evidentes.

## 4. El Plan GlitchBro
Dentro de este cerebro, el plan **GlitchBro** es la línea de estudio prioritaria para el año 2026, centrada en certificaciones técnicas y especialización en Python, Backend y Cloud.
